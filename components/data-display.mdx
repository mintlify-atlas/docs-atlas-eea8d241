---

Data display in Nothing follows an instrument-panel logic: numeric precision is always shown alongside visual proportion, color is applied to the value rather than the label or background, and hierarchy is structural rather than decorative.

## Lists and data rows

Data rows use dividers for structure — never alternating row backgrounds. Labels sit left, values sit right.

| Property | Value |
|----------|-------|
| Divider | `1px solid --border` (`#222222`), full-width |
| Row padding | `12–16px` vertical |
| Label | `Space Mono`, ALL CAPS, `--text-secondary` (`#999999`), left |
| Value | `--text-primary` (`#E8E8E8`), right |
| Alternating rows | Never — use dividers only |

### Stat rows

Stat rows pair a label with a status-colored value and an adjacent unit.

| Element | Style |
|---------|-------|
| Label | `Space Mono`, ALL CAPS, `--text-secondary`, left |
| Value | `Space Mono`, status color (see tokens), right |
| Unit | `--label` size (11px), adjacent to value |
| Trend arrow | Same color as value |

<Note>
  Color is applied to the **value**, never to the label or row background. Labels always remain `--text-secondary`. Trend arrows inherit the value's color.
</Note>

### Hierarchical rows

Sub-items are indented `16–24px` with the same divider treatment as top-level rows. There are no tree lines, expand/collapse controls, or connectors — indentation alone communicates the hierarchy.

```css data-rows.css
/* Row base */
.data-row {
  display: flex;
  align-items: center;
  justify-content: space-between;
  padding: 14px 0;
  border-bottom: 1px solid var(--border); /* #222222 */
}

/* Label — left */
.data-row__label {
  font-family: "Space Mono", monospace;
  font-size: 11px;
  letter-spacing: 0.08em;
  text-transform: uppercase;
  color: var(--text-secondary); /* #999999 */
}

/* Value — right */
.data-row__value {
  font-family: "Space Mono", monospace;
  font-size: 14px;
  color: var(--text-primary);   /* #E8E8E8 */
}

/* Unit — adjacent to value, smaller */
.data-row__unit {
  font-family: "Space Mono", monospace;
  font-size: 11px;
  letter-spacing: 0.08em;
  color: var(--text-secondary);
  margin-left: 4px;
}

/* Stat row — value adopts status color */
.data-row--good    .data-row__value { color: var(--success); /* #4A9E5C */ }
.data-row--caution .data-row__value { color: var(--warning); /* #D4A843 */ }
.data-row--over    .data-row__value { color: var(--accent);  /* #D71921 */ }

/* Hierarchical sub-row */
.data-row--sub {
  padding-left: 20px; /* 16–24px indent */
}
```

## Tables and data grids

Tables use Space Mono for numeric columns and Space Grotesk for text columns. No zebra striping. No cell backgrounds. Active rows use a left accent bar.

| Property | Value |
|----------|-------|
| Header font | `--label` style — `Space Mono`, ALL CAPS, 11px, `--text-secondary` |
| Header border | Bottom `1px solid --border-visible` (`#333333`) |
| Cell padding | `12px 16px` |
| Numeric cells | `Space Mono`, right-aligned |
| Text cells | `Space Grotesk`, left-aligned |
| Zebra striping | Never |
| Cell backgrounds | None |
| Active row | `--surface-raised` bg + left `2px solid --accent` |

<Warning>
  Never add `background-color` to alternating rows. The Nothing table aesthetic relies on clean dividers and the single active-row accent bar. Zebra striping conflicts with the instrument-panel visual language.
</Warning>

```css tables.css
/* Table base */
.data-table {
  width: 100%;
  border-collapse: collapse;
}

/* Header */
.data-table thead th {
  font-family: "Space Mono", monospace;
  font-size: 11px;
  letter-spacing: 0.08em;
  text-transform: uppercase;
  color: var(--text-secondary);          /* #999999 */
  padding: 12px 16px;
  border-bottom: 1px solid var(--border-visible); /* #333333 */
  text-align: left;
}

.data-table thead th.col-numeric {
  text-align: right;
}

/* Cells */
.data-table tbody td {
  padding: 12px 16px;
  border-bottom: 1px solid var(--border); /* #222222 */
  color: var(--text-primary);            /* #E8E8E8 */
  font-family: "Space Grotesk", system-ui, sans-serif;
  font-size: 14px;
}

/* Numeric cells */
.data-table tbody td.col-numeric {
  font-family: "Space Mono", monospace;
  text-align: right;
}

/* Active row — surface-raised bg + 2px left accent */
.data-table tbody tr.is-active td {
  background: var(--surface-raised);     /* #1A1A1A */
}

.data-table tbody tr.is-active td:first-child {
  box-shadow: inset 2px 0 0 var(--accent); /* #D71921 */
}
```

## Segmented progress bars

The signature Nothing data visualization. Discrete rectangular segments read like a hardware meter — mechanical and precise.

### Anatomy

A segmented progress bar consists of a label and value above, followed by a full-width row of discrete rectangular segments with 2px gaps between them.

| Part | Spec |
|------|------|
| Label | `Space Mono`, ALL CAPS, `--text-secondary`, left |
| Value | `Space Mono`, status color, right |
| Segments | Square-ended blocks, no `border-radius` |
| Gap between segments | `2px` |
| Empty segment fill | `--border` (`#222222`) dark / `#E0E0E0` light |

### States

| State | Fill color | Token | When |
|-------|-----------|-------|------|
| Neutral | White / Black | `--text-display` | Within normal range |
| Over limit | Red | `--accent` (`#D71921`) | Exceeds target |
| Good | Green | `--success` (`#4A9E5C`) | Healthy range |
| Moderate | Amber | `--warning` (`#D4A843`) | Caution zone |

### Sizes

| Size | Height | Use |
|------|--------|-----|
| Hero | `16–20px` | Dashboard hero metrics, primary KPIs |
| Standard | `8–12px` | Card content, list rows |
| Compact | `4–6px` | Sparkline-style inline indicators |

<Note>
  Always pair a segmented bar with a numeric readout. The bar shows proportion; the number shows precision. Never use a bar alone without a value label.
</Note>

**Overflow:** When a value exceeds the bar's maximum, filled segments continue past the "full" mark in the active status color. This visually signals the over-limit condition.

```html segmented-bar.html
<div class="seg-bar">
  <div class="seg-bar__header">
    <span class="seg-bar__label">STORAGE USED</span>
    <span class="seg-bar__value seg-bar__value--over">87%</span>
  </div>
  <div class="seg-bar__track" data-segments="20" data-filled="17" data-state="over"></div>
</div>
```

```css segmented-bar.css
/* Bar container */
.seg-bar { width: 100%; }

/* Header row */
.seg-bar__header {
  display: flex;
  justify-content: space-between;
  align-items: baseline;
  margin-bottom: 6px;
}

.seg-bar__label {
  font-family: "Space Mono", monospace;
  font-size: 11px;
  letter-spacing: 0.08em;
  text-transform: uppercase;
  color: var(--text-secondary); /* #999999 */
}

.seg-bar__value {
  font-family: "Space Mono", monospace;
  font-size: 13px;
  color: var(--text-display);   /* neutral default */
}

/* Status colors on value */
.seg-bar__value--good     { color: var(--success); /* #4A9E5C */ }
.seg-bar__value--moderate { color: var(--warning); /* #D4A843 */ }
.seg-bar__value--over     { color: var(--accent);  /* #D71921 */ }

/* Track: flex row of segments */
.seg-bar__track {
  display: flex;
  gap: 2px;
  height: 10px; /* standard size */
}

/* Individual segment — square ends, no border-radius */
.seg-bar__segment {
  flex: 1;
  background: var(--border);   /* #222222 empty */
  border-radius: 0;
}

/* Filled segment — neutral */
.seg-bar__segment--filled {
  background: var(--text-display); /* #FFFFFF */
}

/* Filled segment — states */
.seg-bar--good     .seg-bar__segment--filled { background: var(--success); }
.seg-bar--moderate .seg-bar__segment--filled { background: var(--warning); }
.seg-bar--over     .seg-bar__segment--filled { background: var(--accent);  }

/* Hero size */
.seg-bar--hero .seg-bar__track { height: 18px; }

/* Compact size */
.seg-bar--compact .seg-bar__track { height: 5px; }
```

## Other data visualization

### Bar charts

Bars are vertical, white-filled with `--border` remainder. Square ends — no rounded tops.

| Property | Value |
|----------|-------|
| Fill | `--text-display` (`#FFFFFF`) |
| Empty / remainder | `--border` (`#222222`) |
| Bar ends | Square, `border-radius: 0` |
| Axis labels | `Space Mono`, `--caption` (12px) |
| Grid lines | `--border`, horizontal only |

### Gauges

Thin-stroke circles with tick marks and a centered or adjacent numeric readout.

| Property | Value |
|----------|-------|
| Stroke width | Thin (1–1.5px) |
| Tick marks | Monoline, uniform |
| Readout | Numeric, centered inside or directly adjacent |
| Fill | None (stroke only) |

### Dot grids

Used for heat maps and density visualizations. Dots vary in opacity or size.

```css dot-grid.css
/* Standard dot grid */
.dot-grid {
  background-image: radial-gradient(circle, var(--border-visible) 1px, transparent 1px);
  background-size: 16px 16px;
}

/* Subtle background dot grid */
.dot-grid-subtle {
  background-image: radial-gradient(circle, var(--border) 0.5px, transparent 0.5px);
  background-size: 12px 12px;
  opacity: 0.15;
}
```

### Category differentiation

When multiple data series must be distinguished, apply differentiation in this order — use color as a last resort:

1. **Opacity** — same hue, different opacity levels
2. **Pattern** — hatching, dotting, or segment style
3. **Line style** — solid vs. dashed vs. dotted
4. **Color** — only when the above three are insufficient

### Chart lines

| Element | Spec |
|---------|------|
| Primary line | `1.5–2px`, `--text-display` |
| Average / reference line | `1px` dashed, `--text-secondary` |
| Grid | `--border`, horizontal lines only |
| Area fill | None |
| Legend | None — label lines directly |

```css chart.css
/* Chart line */
.chart-line-primary {
  stroke: var(--text-display); /* #FFFFFF */
  stroke-width: 1.5;
  fill: none;
}

/* Average / reference line */
.chart-line-average {
  stroke: var(--text-secondary); /* #999999 */
  stroke-width: 1;
  stroke-dasharray: 4 4;
  fill: none;
}

/* Axis label */
.chart-axis-label {
  font-family: "Space Mono", monospace;
  font-size: 12px;
  letter-spacing: 0.04em;
  fill: var(--text-secondary); /* #999999 */
}

/* Grid line */
.chart-grid-line {
  stroke: var(--border); /* #222222 */
  stroke-width: 1;
}
```

## Widgets (dashboard cards)

Widgets are surface cards with a hero metric, a category label, and optional instrument motifs.

| Property | Value |
|----------|-------|
| Background | `--surface` (`#111111`) |
| Radius | `16px` |
| Hero metric font | `Doto` or `Space Mono`, large, left-aligned |
| Unit | `--label` size (11px), adjacent to hero metric |
| Category label | `Space Mono`, ALL CAPS, top-left |
| Instrument motifs | Compass, thermometer, dial — thin stroke only |

<Tip>
  Hero metrics use `Doto` (variable dot-matrix) at 36px or larger for maximum instrument-panel impact. Fall back to `Space Mono` when `Doto` is unavailable. Never use `Doto` below 36px.
</Tip>

```css widget.css
/* Widget card */
.widget {
  background: var(--surface);   /* #111111 */
  border: 1px solid var(--border); /* #222222 */
  border-radius: 16px;
  padding: 20px;
}

/* Category label — top-left */
.widget__category {
  font-family: "Space Mono", monospace;
  font-size: 11px;
  letter-spacing: 0.08em;
  text-transform: uppercase;
  color: var(--text-secondary); /* #999999 */
  margin-bottom: 12px;
}

/* Hero metric */
.widget__metric {
  font-family: "Doto", "Space Mono", monospace;
  font-size: 48px;
  line-height: 1.05;
  letter-spacing: -0.02em;
  color: var(--text-display);   /* #FFFFFF */
  display: flex;
  align-items: baseline;
  gap: 4px;
}

/* Unit — adjacent, smaller */
.widget__unit {
  font-family: "Space Mono", monospace;
  font-size: 11px;
  letter-spacing: 0.08em;
  color: var(--text-secondary);
}
```
