This page walks you through triggering the skill, choosing a mode, and getting working UI output from a single prompt.

## The workflow

<Steps>
  <Step title="Trigger the skill">
    In Claude Code, type `/nothing-design` or say "Nothing style" in your message. This activates the skill — Claude loads the design philosophy, token system, and component library.

    <Note>
      The skill never triggers automatically. You must use an explicit trigger phrase.
    </Note>
  </Step>
  <Step title="Claude declares the required fonts">
    Before writing any code, Claude will declare which Google Fonts are needed and how to load them. The Nothing design language requires three fonts:

    - **Space Grotesk** — body and UI text
    - **Space Mono** — data values, ALL CAPS labels
    - **Doto** — hero numbers and display moments only

    Claude provides the `<link>` tag to paste into your HTML `<head>`, or the `@import` for CSS. You do not need to install these fonts manually.
  </Step>
  <Step title="Choose dark or light mode">
    Claude asks which mode to start with. Neither is the default — both are fully designed.

    <Tabs>
      <Tab title="Dark mode">
        Dark mode uses OLED black (`#000000`) as the primary background. Think instrument panel in a dark room — white data glowing against black.

        Key dark mode values:
        - Background: `#000000`
        - Elevated surfaces: `#111111`
        - Body text: `#E8E8E8`
        - Hero text: `#FFFFFF`
        - Labels: `#999999`
      </Tab>
      <Tab title="Light mode">
        Light mode uses warm off-white (`#F5F5F5`) as the primary background. Think printed technical manual — black ink on off-white paper, cards at `#FFFFFF` for subtle elevation without shadows.

        Key light mode values:
        - Background: `#F5F5F5`
        - Elevated surfaces: `#FFFFFF`
        - Body text: `#1A1A1A`
        - Hero text: `#000000`
        - Labels: `#666666`
      </Tab>
    </Tabs>
  </Step>
  <Step title="Describe your UI">
    Tell Claude what you want to build. Be specific about the data or content — the design language does the rest.

    **Example prompt:**

    > Build me a dashboard card showing battery percentage in Nothing style

    Claude will identify the three-layer hierarchy for your UI (primary, secondary, tertiary), apply the spacing and typography rules, and generate the component using exact token values.
  </Step>
  <Step title="Receive your output">
    Claude generates production-ready UI in your target platform. By default it produces HTML/CSS. You can ask for SwiftUI or React/Tailwind instead.
  </Step>
</Steps>

## Example: battery percentage card

Here is the HTML/CSS output for the prompt "Build me a dashboard card showing battery percentage in Nothing style" in dark mode:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Battery Card</title>
  <link rel="preconnect" href="https://fonts.googleapis.com" />
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
  <link href="https://fonts.googleapis.com/css2?family=Space+Grotesk:wght@300;400;500&family=Space+Mono:wght@400;700&family=Doto:ROND@0;100&display=swap" rel="stylesheet" />
  <style>
    :root {
      --black:           #000000;
      --surface:         #111111;
      --border:          #222222;
      --border-visible:  #333333;
      --text-display:    #FFFFFF;
      --text-primary:    #E8E8E8;
      --text-secondary:  #999999;
      --text-disabled:   #666666;
      --accent:          #D71921;
      --success:         #4A9E5C;
      --warning:         #D4A843;

      --space-xs:  4px;
      --space-sm:  8px;
      --space-md:  16px;
      --space-lg:  24px;
      --space-xl:  32px;
      --space-2xl: 48px;
    }

    * {
      margin: 0;
      padding: 0;
      box-sizing: border-box;
    }

    body {
      background: var(--black);
      display: flex;
      align-items: center;
      justify-content: center;
      min-height: 100vh;
      font-family: "Space Grotesk", "DM Sans", system-ui, sans-serif;
    }

    .card {
      background: var(--surface);
      border: 1px solid var(--border-visible);
      border-radius: 8px;
      padding: var(--space-xl);
      width: 280px;
    }

    /* Tertiary: instrument label */
    .card-label {
      font-family: "Space Mono", "JetBrains Mono", monospace;
      font-size: 11px;
      letter-spacing: 0.08em;
      text-transform: uppercase;
      color: var(--text-secondary);
      margin-bottom: var(--space-lg);
    }

    /* Primary: hero number */
    .battery-value {
      font-family: "Doto", "Space Mono", monospace;
      font-size: 72px;
      line-height: 1.0;
      letter-spacing: -0.03em;
      color: var(--text-display);
      margin-bottom: var(--space-xs);
    }

    .battery-unit {
      font-family: "Space Mono", monospace;
      font-size: 11px;
      letter-spacing: 0.08em;
      text-transform: uppercase;
      color: var(--text-secondary);
      vertical-align: super;
      margin-left: var(--space-xs);
    }

    /* Segmented progress bar */
    .progress-track {
      display: flex;
      gap: 3px;
      margin-top: var(--space-lg);
      margin-bottom: var(--space-md);
    }

    .segment {
      flex: 1;
      height: 4px;
      background: var(--border-visible);
      border-radius: 0; /* square ends — no radius */
    }

    .segment.filled {
      background: var(--text-display);
    }

    /* Secondary: status row */
    .status-row {
      display: flex;
      justify-content: space-between;
      align-items: baseline;
    }

    .status-label {
      font-family: "Space Mono", monospace;
      font-size: 11px;
      letter-spacing: 0.08em;
      text-transform: uppercase;
      color: var(--success);
    }

    .status-time {
      font-family: "Space Mono", monospace;
      font-size: 11px;
      letter-spacing: 0.08em;
      color: var(--text-disabled);
    }
  </style>
</head>
<body>
  <div class="card">
    <div class="card-label">Battery</div>

    <div class="battery-value">
      78<span class="battery-unit">%</span>
    </div>

    <div class="progress-track">
      <!-- 10 segments, 8 filled = 78% rounded to nearest segment -->
      <div class="segment filled"></div>
      <div class="segment filled"></div>
      <div class="segment filled"></div>
      <div class="segment filled"></div>
      <div class="segment filled"></div>
      <div class="segment filled"></div>
      <div class="segment filled"></div>
      <div class="segment filled"></div>
      <div class="segment"></div>
      <div class="segment"></div>
    </div>

    <div class="status-row">
      <span class="status-label">Charging</span>
      <span class="status-time">4H 12M remaining</span>
    </div>
  </div>
</body>
</html>
```

The output demonstrates the three-layer rule in practice:

- **Primary** — `78` in Doto at 72px (`--display-xl`), `--text-display` white
- **Secondary** — the segmented progress bar, showing proportion visually
- **Tertiary** — `BATTERY` label and status row in Space Mono ALL CAPS, `--text-secondary` and `--text-disabled`

<Tip>
  The segmented progress bar is a core Nothing component. Claude always builds it with discrete segments and a precise gap, never with a smooth filled bar.
</Tip>
